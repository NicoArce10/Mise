# Mise — Milestone 3 execution plan — Backend shell

> Produced by `writing-plans` on 2026-04-23.
> Anchors to `docs/plans/2026-04-22-architecture.md` §2.3, §3, §5 (T3.1–T3.10), and to `3rd_prompt.md`.
> No AI calls in this milestone. Opus 4.7 integration is Milestone 4.

## Goal
Scaffold a FastAPI service that matches the API contract from Milestone 1 and returns mock responses shaped exactly like the real ones. The Cockpit must consume this backend without any changes to its domain types.

## Architecture (one line)
Four API endpoints + one internal state machine, backed by a process-local in-memory store keyed by `processing_id`. Fixtures mirror the frontend mock UUID-for-UUID so the Cockpit swaps `fetch` URLs in place.

## Tech stack
- Python 3.11+
- FastAPI 0.115+
- Pydantic v2.9+
- uvicorn 0.32+
- pytest 8+
- python-multipart 0.0.12+ (for upload)
- httpx 0.28+ (for TestClient)
- Absolutely no LangChain, LlamaIndex, supabase-py, openai, or OCR libraries

## Tasks (bite-sized, 20–45 min each)

### T3.1 — Scaffold FastAPI
Files: `backend/requirements.txt`, `backend/pyproject.toml`, `backend/app/__init__.py`, `backend/app/main.py`, `backend/app/core/__init__.py`, `backend/app/core/config.py`.
- Pin: fastapi, pydantic, pydantic-settings, uvicorn[standard], python-multipart, httpx, pytest
- `pyproject.toml`: ruff config (line-length 100, target py311, select E/F/I/N/UP/B), pytest config (testpaths=["tests"])
- `config.py`: `Settings(BaseSettings)` with `anthropic_model`, `api_host`, `api_port`, `cors_origins`; reads from `.env` at repo root via `model_config = SettingsConfigDict(env_file=Path(__file__).parent.parent.parent / ".env", extra="ignore")`
- `main.py`: `create_app()` factory returning `FastAPI(title="Mise", version="0.1.0")`. CORS middleware from config. No routers yet (T3.4+ adds them).

**Acceptance:** `uvicorn app.main:app --reload --port 8000` boots, `GET /` returns 404 (no root route), `GET /docs` renders Swagger.

### T3.2 — Domain models
File: `backend/app/domain/__init__.py`, `backend/app/domain/models.py`.
- Paste architecture plan §2.3 verbatim with the flat-modifiers fix already applied
- `from __future__ import annotations` at the top
- Enums are `str, Enum`
- `ConfigDict(frozen=True)` on value objects (`SourceDocument`, `EvidenceRecord`)
- All field ordering: types → defaults → Field(...) constraints

**TDD:** `test_models.py` does `model.model_validate(model.model_dump(mode="json"))` round-trip for every class. All 12+ classes.

**Acceptance:** `pytest backend/tests/test_models.py -q` passes.

### T3.3 — Fixtures mirroring frontend
File: `backend/app/domain/fixtures.py`.
- Export one function `load_cockpit_fixture() -> CockpitState`
- UUIDs are identical to `frontend/src/mocks/cockpit.ts` IDS constant. Use `UUID(hex=...)` or string-cast where the frontend uses string IDs like `'src-pdf-branch-a'`. The cleanest path: change the frontend Ids to actual UUID hex strings in T3.10, OR accept string-based IDs on the backend since the Pydantic type is `UUID` which coerces `uuid.UUID(str)`. **Decision:** the frontend currently uses human-readable string IDs (`'dish-margherita'`). These are NOT valid UUIDs. The backend will use proper UUID4 values; T3.10 updates the frontend mock IDs to match.
- Every decision text copied verbatim from the frontend mock (exact strings)
- All four demo-critical decisions present: Margherita merge, Pizza Funghi vs Calzone Funghi non-merge, burrata modifier, Chef's Special ephemeral
- Plus the `add guacamole +2` unattached modifier (for the bundle 02 demo path)

**Acceptance:** `load_cockpit_fixture()` returns a valid `CockpitState` that `.model_dump_json()` serializes cleanly.

### T3.4 — In-memory store
File: `backend/app/core/store.py`.
- `class InMemoryStore` with dict-backed state, protected by a `threading.Lock`
- Keys: `batches: dict[UUID, UploadBatch]`, `runs: dict[UUID, CockpitState]`, `run_meta: dict[UUID, ProcessingRun]`
- Module-level singleton `store = InMemoryStore()`
- Methods: `save_batch`, `get_batch`, `create_run_from_batch(batch_id) -> UUID`, `get_run_state(run_id)`, `get_cockpit(run_id)`, `apply_decision(run_id, decision)`
- `UploadBatch` is NOT in §2.3 — define it here as a small internal model: `{ id: UUID, sources: list[SourceDocument], uploaded_at: str }`

**Acceptance:** unit test inserts a batch, creates a run, reads back the cockpit — round-trip clean.

### T3.5 — Upload endpoint (POST /api/upload)
File: `backend/app/api/__init__.py`, `backend/app/api/upload.py`.
- Accept `files: list[UploadFile] = File(...)`
- Validate: at most 10 files, each ≤ 10 MB, `content_type in {"image/jpeg","image/png","application/pdf"}`
- For each: read bytes, compute SHA256, infer `SourceKind` from content_type + filename, extract dimensions for images via Pillow (add to requirements.txt) OR skip dimensions in MVP and set to None — **Decision: skip Pillow, set width_px/height_px to None**. Keeps the MVP tight; dimensions are not used by the Cockpit renderer today.
- Return `201 { "batch_id": UUID, "sources": [SourceDocument] }`
- **Mock-mode short-circuit:** if filename matches any fixture source filename (`menu_pdf_branch_a.pdf`, etc.), use the fixture UUID so downstream processing returns the fixture. This is how the demo stays deterministic.

**TDD:** `test_upload.py`: POST valid multipart → 201 with batch_id. POST empty → 400. POST non-image/pdf → 415.

### T3.6 — Processing start (POST /api/process/{batch_id})
File: `backend/app/api/processing.py`.
- `POST /api/process/{batch_id}` → create a `ProcessingRun` with `state=QUEUED`, schedule a background task via `BackgroundTasks` that advances state through `EXTRACTING → RECONCILING → ROUTING → READY` with 250ms sleeps, populating the cockpit from fixtures when ready.
- Return `202 { "processing_id": UUID }`
- If `batch_id` unknown → 404

**TDD:** `test_processing.py` part A: POST unknown batch → 404. POST valid batch → 202 with uuid.

### T3.7 — Processing status (GET /api/process/{processing_id})
- Return current `ProcessingRun` with `state_detail` = "Adaptive thinking engaged on N pairs" when state == RECONCILING and N > 0 (fixture says 2). Otherwise null.

**TDD:** `test_processing.py` part B: POST process → poll 10 times with 100ms sleeps → last state is `ready`. Counts transitions, asserts `state_detail` contains "Adaptive thinking" at some point.

### T3.8 — Review GET (GET /api/review/{processing_id})
File: `backend/app/api/review.py`.
- Return the full `CockpitState` from the store.
- If run not found → 404.
- If run not yet READY → return a partial CockpitState (state reflects current stage; `canonical_dishes=[]`, `modifiers=[]`, `ephemerals=[]`, `reconciliation_trace=[]`).

**TDD:** `test_review.py`: POST upload → POST process → wait for ready → GET review → assert all four demo-critical decisions present by canonical_name / text / lead_word. Assert `modifiers` includes `add burrata +3` with `parent_dish_id` = Margherita's id. Assert `Chef's Special` in ephemerals.

### T3.9 — Decisions POST (POST /api/review/{processing_id}/decisions)
- Body: `{ target_kind: "canonical"|"modifier"|"ephemeral", target_id: UUID, action: "approve"|"edit"|"reject", edit?: object }`
- Mutates the in-memory state: sets `moderation = approved/edited/rejected` on the target entity.
- For `action="edit"`, apply partial overrides to canonical_name/aliases/ingredients on canonical, or text on modifier/ephemeral.
- Returns the full updated `CockpitState` so the frontend re-renders.
- If target id not found → 404; invalid action → 422.

**TDD:** `test_review.py` part B: approve Margherita → GET review → Margherita.moderation == approved.

### T3.10 — Wire frontend to backend
Files: `frontend/src/api/client.ts` (new), `frontend/src/hooks/useCockpitState.ts` (extend), `frontend/src/mocks/cockpit.ts` (update IDs to UUID format OR keep string IDs and align backend fixture).
- Create `frontend/src/api/client.ts` with typed `apiUpload`, `apiStartProcessing`, `apiGetProcessing`, `apiGetReview`, `apiPostDecision` using `import.meta.env.VITE_API_BASE`.
- `useCockpitState`: accept a prop `source: 'mock' | 'api'`. When `api`, poll `/api/process/{id}` every 750 ms until READY, then fetch `/api/review/{id}`.
- App.tsx: detect `VITE_API_BASE` at build time. If set, default to `source='api'`. Fall back to `'mock'` if fetch throws (for offline demos).
- **Re-sync mock IDs with backend fixture UUIDs.** Either update both to share the same UUID v4 strings, or update the frontend to use hex-safe 36-char strings. Decision: both sides use the SAME human-readable strings as non-UUID IDs, and the Pydantic models accept `str | UUID` via a custom type alias `EntityId = str`. This avoids UUID formatting friction and keeps the mock readable.
  - Change architecture plan §2.3 `id: UUID` → `id: str` across Pydantic models. Validation: Pydantic still enforces `str`. This is a second documented extension.

**Acceptance:** Launch backend + frontend. Visit `127.0.0.1:5173`. Click "New batch" → upload any files → Cockpit renders all 4 demo-critical decisions from the backend. Click "Present" → hero frame works with backend-sourced data.
