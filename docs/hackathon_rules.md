# Hackathon Rules and Guardrails

## Official constraints
- Everything shown in the demo must be fully open source
- All work must be started from scratch during the hackathon
- Team size max 2
- Do not use code, data, or assets without the rights to use them
- Submission deadline: Sunday, April 26, 8:00 PM EST

## Submission requirements
- 3-minute demo video maximum
- Open source repo / code link
- Written summary

## Judging implications
Stage 1 is asynchronous.
Judges will review:
- the video
- the repo
- the written summary

This means the project must be understandable without live explanation.

## Judging criteria
1. Impact
2. Demo
3. Opus 4.7 use
4. Depth and execution

## Project guardrails for Mise
- Mise must be clearly standalone and new
- Do not reference any prior product anywhere in code, UI, docs, prompts, or assets
- Do not reuse pre-existing code
- Do not connect to pre-existing databases
- Do not use risky restaurant assets unless clearly licensed or created by us
- Prefer synthetic but highly realistic demo assets
- The core product must work without Managed Agents
- Managed Agents can only be a bonus, not a dependency for the MVP

## Strategy
To maximize judging outcomes:
- prioritize a visually strong demo
- keep the product easy to understand in under 20 seconds
- show clear before/after transformation
- show why Opus 4.7 matters
- avoid features that do not improve the demo
