# Evaluation Datasets

Three synthetic restaurant bundles, all authored for this hackathon. No external or real restaurant assets are used. Every file here must be either generated by us or derived from assets we created.

## Asset specification per bundle

### Bundle 01 — Italian trattoria (three branches)
Purpose: carry the `Marghertia -> Margherita` normalization case and the `Pizza Funghi != Calzone Funghi` non-merge case.

Assets to author:
- **`menu_pdf_branch_a.pdf`** — a one-page PDF menu for "Branch A". Must contain: `Marghertia` (misspelled on purpose), `Pizza Funghi`, `Pizza Diavola`. Add the typo exactly as shown. Generate using any document tool and export to PDF.
- **`menu_photo_branch_b.jpg`** — a photograph-style image (square, ~1200×1200) of a printed menu. Must contain: `Margherita` (spelled correctly), `Calzone Funghi`, `Calzone Vegano`. Use a legible display font. Add a slight warm paper tint so it reads as a photo, not a screenshot.
- **`chalkboard_branch_c.jpg`** — a chalkboard-style image. Must contain: `Margherita` with the annotation `add burrata +3`, and `Pizza Diavola` with `extra chili +1`. Handwritten-style font acceptable as long as it is legible to a human.
- **`instagram_post_special.png`** — a square social-post image. Must contain a dish "Linguine del giorno" presented as a daily special, no canonical name alignment across the other sources. This exercises the ephemeral routing case in addition to bundle 03.

### Bundle 02 — Taqueria
Purpose: carry the `add guacamole +2` modifier case.

Assets to author:
- **`menu_pdf_main.pdf`** — one-page PDF with: `Tacos al Pastor`, `Tacos de Carnitas`, `Tacos de Barbacoa`, `Quesadilla de Queso`.
- **`menu_screenshot_delivery.png`** — a mock delivery-platform screenshot (iOS-shaped frame acceptable). Must contain the same four dishes with slight name variants: `Al Pastor Tacos`, `Carnitas Tacos`, `Barbacoa Tacos`, `Cheese Quesadilla`. This exercises reconciliation across sources with re-ordered token names.
- **`modifiers_chalkboard.jpg`** — a chalkboard image listing modifiers only: `add guacamole +2`, `add queso +1`, `extra salsa +0`. Must NOT resemble a dish list. This is the ambiguous case the router must treat as modifiers.

### Bundle 03 — Modern bistro
Purpose: carry the `Chef's special` ephemeral case.

Assets to author:
- **`menu_pdf_dinner.pdf`** — dinner menu with: `Short Rib`, `Halibut en Papillote`, `Beet Tartare`, `Ricotta Gnudi`.
- **`menu_pdf_lunch.pdf`** — lunch menu with partial overlap: `Beet Tartare`, `Ricotta Gnudi`, `Mushroom Toast`. Shared items must reconcile to the same canonical dish across lunch and dinner.
- **`chef_special_board.jpg`** — a hand-lettered daily board with `Chef's special — ask your server`. No stable name, no fixed price. Must be routed as ephemeral, never as a canonical dish.

## Authoring guidance

Use synthetic tools only. Acceptable sources for asset generation:
- Typography and layout in a word processor / Figma, exported to PDF or PNG
- An image model (any open tool) with the dish names and "photograph of a printed restaurant menu, plain warm paper" as the brief
- Hand-drawn or hand-typed chalkboard images using a chalkboard SVG background we created

Forbidden:
- Pulling menus from real restaurants
- Using real restaurant names or logos
- Stock imagery unless its license is explicitly permissive and attribution is added to this README

## `expected.json` shape

Every bundle ships an `expected.json`. The schema lives in `docs/evals.md`. Copy that schema verbatim when authoring each file. Every field is required; use empty arrays when no cases apply.

## What the harness does with these

`python evals/run_eval.py --bundle all` reads each bundle, feeds the evidence files to the pipeline, and compares the output to `expected.json`. Metrics are computed per bundle and aggregated. The generated report lands in `evals/reports/` and is gitignored.

The numbers that appear in the demo video and the written summary come from one specific run of this harness. That run's report is copied to `submissions/metrics.json` on submission day. No other source of numbers is allowed.
